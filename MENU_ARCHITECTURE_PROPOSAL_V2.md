# 菜单架构优化建议 V2 - 双系统独立视角

**分析日期：** 2025-11-18
**重要前提：** 平台后台和酒店后台是**两个独立系统**，不同域名，不同用户

---

## 🎯 关键理解修正

### 系统架构澄清

```
平台后台系统（admin.platform.com）
├─ 用户：平台运营人员
├─ 权限：查看所有酒店数据
└─ 职责：B端招商、C端运营、平台治理

酒店后台系统（hotel.platform.com）
├─ 用户：酒店管理员/前台
├─ 权限：只看自己酒店数据
└─ 职责：门店运营、订单处理、收益管理
```

**现在放在一起只是为了学习展示！**

### 会员管理的特殊性

```
平台后台：
- 设定会员等级（VIP1/VIP2/VIP3）
- 设定最低折扣（如VIP3最低9折）
- 查看全平台会员数据

酒店后台：
- 查看本店会员消费数据（只读）
- 在平台折扣基础上上浮（如VIP3在本店8.5折）
- 设定本店会员专属权益
```

**结论：** 两个系统都应该有会员管理，但功能不同！

### 订单管理的双重性

```
平台后台：
- 查看所有酒店订单
- 监控平台GMV
- 处理平台级退款仲裁

酒店后台：
- 只看本店订单
- 处理日常订单（确认、分配、入住、退房）
- 处理商家侧退款审核
```

**结论：** 两个系统都应该有订单管理，但视角不同！

## 🔍 关键设计问题

### 问题1：会员管理应该如何设计？

**方案1：两个系统都有会员管理（推荐）✅**

```
平台后台 - 会员管理：
├─ 会员等级配置          # 设定VIP1/2/3规则
├─ 等级折扣配置          # VIP1最低9.5折，VIP2最低9折
├─ 会员列表              # 查看全平台所有会员
└─ 会员权益管理          # 平台级权益（生日礼、升级奖励）

酒店后台 - 会员运营（NEW）：
├─ 本店会员列表          # 查看在本店消费过的会员（基于订单）
├─ 会员折扣设置          # 在平台最低折扣基础上上浮
│  └─ 例：VIP3平台最低9折，本店可设8.5折
└─ 店内会员权益          # 本店专属：生日房、周年礼
```

**业务逻辑：**
```
最终折扣 = min(平台最低折扣, 酒店设定折扣)

例子：
- VIP3会员，平台规定最低9折
- A酒店设置VIP3享8.5折 → 用户在A酒店享8.5折 ✓
- B酒店设置VIP3享9.2折 → 用户在B酒店享9折（不能低于平台最低） ✓
```

**方案2：只平台有会员管理，酒店只读（之前的方案）❌**
```
问题：酒店无法做会员运营，无法留住常客
```

### 问题2：订单管理应该如何设计？

**推荐：两个系统都有，但功能不同**

```
平台后台 - 订单管理：
├─ 订单列表              # 查看所有酒店订单（监控GMV）
├─ 订单统计              # 各酒店销量排名
├─ 退款管理              # 平台级退款概览
└─ 异常订单监控          # 超时未确认、纠纷订单

酒店后台 - 订单处理：
├─ 订单列表              # 只看本店订单
├─ 订单日历              # 本店入住安排
├─ 客诉退款              # 处理本店退款申请
└─ 用户评价              # 本店评价回复
```

**数据隔离：**
- 平台：`SELECT * FROM orders`（全部）
- 酒店：`SELECT * FROM orders WHERE hotel_id = ?`（仅本店）
