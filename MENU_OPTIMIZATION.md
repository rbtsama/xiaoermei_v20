# 菜单结构优化记录

## ✅ 优化内容

### 删除的菜单项

1. **酒店后台 - 会员管理** ❌ 已删除
   - **原因：** 根据#002修复方案，会员统一由平台管理，酒店后台不再管理会员
   - **路径：** `/hotel-backend/members`

2. **平台后台 - 会员权益管理** ❌ 已删除
   - **原因：** 没有对应的页面实现
   - **路径：** `/member/benefits`

3. **平台后台 - 订单详情（独立菜单）** ❌ 已删除
   - **原因：** 通过订单列表点击进入即可，不需要独立菜单项
   - **路径：** `/order/detail`

4. **平台后台 - 营销管理** ❌ 已删除
   - **原因：** 只有一个"广告管理"页面，内容太少，合并到其他分类更合理
   - **路径：** `/marketing/ads`

5. **平台后台 - 账号管理（独立分类）** ❌ 已删除
   - **原因：** 只有一个页面，合并到"系统管理"更合理

### 新增/优化的菜单项

1. **平台后台 - 系统管理** ✅ 新增分类
   - 合并了账号、用户、协议、标签管理
   - 统一管理系统级配置

2. **酒店后台 - 系统对接** ✅ 新增分类
   - PMS对接
   - 房间管理
   - 员工账号
   - 归类更合理

3. **简化名称** ✅
   - "协议模板管理" → "协议模板"
   - "用户点赞" → "用户评价"
   - "会员体系" → "会员管理"

## 🎯 菜单设计原则

### 1. 业务聚焦
- 平台后台：运营管理（积分、会员、优惠券、酒店、订单、争议）
- 酒店后台：日常运营（门店、房型、房务、系统对接）

### 2. 层级合理
- 最多三级菜单
- 一级分类不超过8个
- 二级分类页面2-7个为宜

### 3. 避免重复
- 会员管理只在平台后台
- 订单管理平台和酒店各有，但数据隔离

### 4. 删除冗余
- 内容太少的分类合并
- 通过列表进入的详情页不需要独立菜单
