# 门店部署 PRD文档

## 一、功能概述

### 1.1 功能目标

为店长提供一个清晰、连贯的酒店信息录入系统，支持分步填写、自动保存草稿，确保信息完整准确地提交给平台审核。

### 1.2 用户画像

- 酒店/民宿店长或运营人员
- 互联网操作能力中等
- 可能需要多次填写才能完成
- 需要在填写过程中查找和准备素材

### 1.3 核心功能

- 分Tab分步填写表单
- 自动保存草稿功能
- 图片批量上传与预览
- 房型信息管理（添加、编辑、删除、复制）
- 提交审核

## 三、整体流程与Tab结构

### 3.1 Tab列表

| Tab序号 | Tab名称        | 包含内容                                                     |
| ------- | -------------- | ------------------------------------------------------------ |
| Tab 1   | 账号与门店信息 | 账号信息、门店基本信息、门店介绍                             |
| Tab 2   | 设施与周边     | 门店设施、周边信息（交通/景点/逛吃）                         |
| Tab 3   | 运营政策       | 预订/入住/退房政策、取消政策、押金政策、早餐政策、支付方式等 |
| Tab 4   | 门店展示       | 门店亮点、所有图片、视频素材                                 |
| Tab 5+  | 房型配置       | 动态生成，根据房间数量添加多个房型                           |

### 3.2 Tab导航规则

- 允许跳转到任意Tab，不强制按顺序填写
- 未完成必填项的Tab显示未完成状态
- 已完成所有必填项的Tab显示完成状态
- 当前正在编辑的Tab高亮显示

### 3.3 Tab状态显示

- 完成状态：Tab名称后显示 "✓"
- 未完成状态：Tab名称后显示 "未完成" 或 "必填项未完成"
- 当前Tab：高亮显示
- 选填内容：在具体字段处标注"选填"

## 五、全局交互规则

### 5.1 自动保存机制

**触发条件：**

- 用户在任何输入框完成输入后5秒
- 用户切换Tab时
- 用户点击"保存草稿"按钮时

**保存提示：**

- 顶部固定显示保存状态条
- 保存中："正在保存..."
- 保存成功："草稿已自动保存 HH:MM"
- 保存失败："保存失败，请检查网络"

**草稿管理：**

- 系统仅保留1份草稿
- 用户重新进入时，如有未完成的草稿，显示提示："您有未完成的申请草稿，上次编辑时间：YYYY-MM-DD HH:MM"
- 提供两个按钮：[继续填写] [重新开始]
- 点击"重新开始"需二次确认："确认重新开始？这将清空所有已填写内容，此操作不可恢复。"

### 5.2 页面离开提示

**触发条件：**

- 用户有未保存的修改
- 用户尝试关闭页面、刷新页面或跳转到其他页面

**提示内容：** "您有未保存的修改，确认离开吗？草稿已自动保存，您可以随时回来继续填写。"

- [确认离开]
- [继续编辑]

### 5.3 Tab切换规则

**允许自由切换：**

- 所有Tab可随时点击跳转，不强制按顺序填写
- 切换前自动保存当前Tab内容

**Tab状态标识：**

- 完成状态：Tab名称后显示绿色 "✓"
- 未完成状态：Tab名称后显示 "未完成"
- 当前Tab：高亮显示

**提交前校验：**

- 用户点击"提交审核"时，检查所有必填项是否完成
- 如有未完成的必填项，弹窗提示："以下Tab还有必填项未完成：Tab X、Tab Y，请完成后再提交"，并提供[前往填写]按钮

### 5.4 表单校验规则

**实时校验：**

- 手机号：输入时实时校验格式，格式错误时在输入框下方显示红色提示"请输入正确的手机号格式"
- 房间数量：房型总数自动累加，实时提示"已配置X间，还需配置X间"或"已配置X间，超出门店总数X间"
- 字数限制：显示实时字数统计"X/1000字"，超出时标红提示"已超出字数限制"

**提交时校验：**

- 所有必填字段不能为空
- 格式校验（手机号、时间、数字等）
- 图片上传数量和大小校验
- 房型总数必须等于门店总房间数
- 多选必填项必须至少选择规定数量

**错误提示：**

- 字段下方显示红色错误文字
- 页面顶部显示错误汇总："表单填写有误，请检查标红字段"

### 5.5 图片上传交互

**上传方式：**

- 点击上传区域选择文件
- 拖拽文件到上传区域

**上传过程：**

- 显示上传进度："上传中... 50%"
- 上传完成后显示预览图
- 预览区域比例与要求比例一致

**预览与编辑：**

- 显示图片缩略图
- 显示文件名和文件大小
- 提供[重新上传]和[删除]按钮
- 如系统自动居中裁剪，提示："系统已自动居中裁剪，如效果不理想，请自行调整图片后重新上传"

**批量上传：**

- 支持一次选择多个文件
- 显示所有已上传图片
- 支持拖拽调整图片顺序
- 显示已上传数量："已上传 3/5 张"

### 5.6 房型复制功能

**触发位置：**

- 房型列表页，每个房型卡片显示[复制]按钮

**复制逻辑：**

1. 点击[复制]按钮
2. 系统创建一个新房型，完全复制原房型的所有信息
3. 房型名称自动添加"-副本"后缀（如"山景大床房-副本"）
4. 跳转到房型编辑页，店长可修改需要调整的字段
5. 保存后，新房型添加到房型列表

**说明：**

- 复制不影响原房型
- 复制后的房型独立存在，修改互不影响

## 六、提交与审核

### 6.1 提交条件

**必须满足以下条件才能提交：**

1. 所有标记为"必填 ★"的字段都已完成
2. 所有房型的数量总和等于门店总房间数
3. 至少配置了1个房型
4. 所有必填图片已上传
5. 所有必填的多选项至少选择了规定数量

**检查逻辑：**

- 用户点击"提交审核"按钮时进行全量检查
- 如有未完成项，弹窗提示具体内容："以下内容未完成：
  - Tab1：主账号未填写
  - Tab2：门店设施至少需要选择3项
  - Tab5：房型总数为18间，门店总房间数为21间，还需配置3间"
- 提供[前往填写]按钮，点击后跳转到第一个未完成的Tab

### 6.2 提交确认

**提交前确认弹窗：** "请确认信息无误后提交审核。提交后将无法修改，平台将在3个工作日内完成审核。"

- [再检查一下] - 取消提交
- [确认提交] - 提交审核

### 6.3 提交成功

**提示页面：**

```
✓ 提交成功！

您的申请已提交，平台将在3个工作日内完成审核。
审核结果将通过短信和站内消息通知您。

申请编号：20241210001
提交时间：2024-12-10 15:30

[查看审核进度] [返回首页]
```

### 6.4 审核状态

**审核状态类型：**

- 待审核：申请已提交，等待平台审核
- 审核中：平台正在审核
- 审核通过：申请通过，门店已上线
- 审核驳回：申请未通过，需修改后重新提交

**审核驳回处理：**

- 显示驳回原因（具体到字段或Tab）
- 允许修改草稿
- 修改后重新提交审核

## 七、特殊场景处理

### 7.1 网络异常

**场景1：自动保存失败**

- 顶部显示："保存失败，请检查网络连接"
- 保留用户当前填写内容
- 网络恢复后自动重试保存

**场景2：图片上传失败**

- 显示上传失败的图片："上传失败，请重试"
- 提供[重新上传]按钮
- 不影响其他已上传的图片

**场景3：提交失败**

- 提示："提交失败，请检查网络后重试"
- 保留所有填写内容
- 提供[重新提交]按钮

### 7.2 数据异常

**场景1：房型数量不匹配**

- 实时提示："所有房型数量总和为X间，门店总房间数为Y间，请确保数量一致"
- 提交时再次校验，不一致则不允许提交，并明确提示差异

**场景2：必填项遗漏**

- 提交时弹窗列出所有未完成的必填项
- 按Tab分组显示
- 提供快速跳转功能

**场景3：重点展示超限**

- 周边信息重点展示勾选超过2项时，禁用其他选项
- 提示："最多只能选择2项重点展示，已选择X/2项"
